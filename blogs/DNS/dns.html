DNS:-

     DNS, also known as Domain Name System, is a diverse process without which using the internet is not possible. DNS is used to translate the the url entered by the user to it's corresponding IP to let the web browser know where the user wants to navigate.


There are 4 DNS servers involved in loading a web page:-

      1. DNS recursor
      2. Root nameserver
      3. TLD nameserver
      4. Authoritative nameserver


1. DNS recursor:-

It can be thought of as a librarian who we ask to get a certain book. It receives the queries from client machines through applications such as a web browser. It is responsible for making additional requests in order to satisfy the client's DNS query.

2. Root nameserver:-

It is the first step in translating/resolving the human readable host names into IP addresses It can be thought of an index in library that points to racks of books.

3. TLD nameserver:-

The top level domain can be thought of the rack of books in a library. This nameserver is the next step in search for a specific IP address, it hosts the last portion of a domain name. For example in (google.com), the TLD server is "com".

4. Authoritative nameserver:-

It can be thought of as a dictionary in the rack of books. It is responsible for translating the domain name to it's corresponding IP address (if exists), and returns the IP to the DNS recursor.

DNS Lookup:-

There are 8 steps in a DNS lookup:-

1. A user types ‘example.com’ into a web browser and the query travels into the Internet and is received by a DNS recursive resolver.

2. The resolver then queries a DNS root nameserver (.).

3. The root server then responds to the resolver with the address of a Top Level Domain (TLD) DNS server (such as .com or .net), which stores the information for its domains. When searching for example.com, our request is pointed toward the .com TLD.

4. The resolver then makes a request to the .com TLD.

5. The TLD server then responds with the IP address of the domain’s nameserver, example.com.

6. Lastly, the recursive resolver sends a query to the domain’s nameserver.

7. The IP address for example.com is then returned to the resolver from the nameserver.

8. The DNS resolver then responds to the web browser with the IP address of the domain requested initially.

Types of query:-

1. Recursive query:-

In a recursive query, a DNS client provides a hostname, and the DNS Resolver “must” provide an answer—it responds with either a relevant resource record, or an error message if it can't be found. The resolver starts a recursive query process, starting from the DNS Root Server, until it finds the Authoritative Name Server (for more on Authoritative Name Servers see DNS Server Types below) that holds the IP address and other information for the requested hostname.

2. Iterative query:-

In an iterative query, a DNS client provides a hostname, and the DNS Resolver returns the best answer it can. If the DNS resolver has the relevant DNS records in its cache, it returns them. If not, it refers the DNS client to the Root Server, or another Authoritative Name Server which is nearest to the required DNS zone. The DNS client must then repeat the query directly against the DNS server it was referred to.

3. Non-Recursive query:-

A non-recursive query is a query in which the DNS Resolver already knows the answer. It either immediately returns a DNS record because it already stores it in local cache, or queries a DNS Name Server which is authoritative for the record, meaning it definitely holds the correct IP for that hostname. In both cases, there is no need for additional rounds of queries (like in recursive or iterative queries). Rather, a response is immediately returned to the client.

DNS caching:-

The purpose of caching is to improve performance of data requests by the user. DNS data can be cached in a variety of locations each of which store DNS records for a set time determined by TTL (Time To Live).

Browser DNS caching:-

Nowadays, browsers are designed by default to cache DNS records for a set amount of time. The closer the FNS caching occurs to the web browser, the fewer processing steps must be taken to get the IP. The browser cache is the first location checked for the requested record.

OS level DNS caching:-

The operating system level DNS resolver is the second and last local stop before a DNS query leaves your machine. The process inside your operating system that is designed to handle this query is commonly called a “stub resolver” or DNS client. When a stub resolver gets a request from an application, it first checks its own cache to see if it has the record. If it does not, it then sends a DNS query (with a recursive flag set), outside the local network to a DNS recursive resolver inside the Internet service provider (ISP).

DNS records:-

They hold information related to your domain name. The types of DNS records are as follows:-

      A, AAAA, CNAME, MX, TXT, NS

A record:-

A record stands for Address record and it refers to the ip address of the server to which this domain points. TTL means for how much time, the server wants to cache the ip address on the client's system. An example of how the A record may look like is:-
TYPE:- A
NAME:- @ (@ means it is not a sub domain like etc.roadmap.sh || root domain)
VALUE:- 185.199.108.153 (IP)
TTL:- 14400 (seconds)

Another example:-

TYPE:- A
NAME:- app (app means a subdomain of app.roadmap.sh || can also be "your-site.com")
VALUE:- 185.199.108.153 (IP)
TTL:- 14400 (seconds)

CNAME record:-

Stands for Canonical name record. Points a domain or subdomain to another domain.

An example is:-
TYPE:- CNAME
NAME:- www
VALUE:- roadmap.sh
TTL:- 14400 (seconds)
It returns the IP for roadmap.sh
Here, roadmap.sh is the A record and www is the CNAME. CNAME IS simple a name pointing to a domain. (like docs.google.com points to google.com) CNAME IS USED TO REDIRECT.

MX record:-

Stands for Mail Exchanger. It helps route the emails to the relevant server. (Points to email hosting providers like :- G-SUITE, Office365, Exchange. G-SUITE and ZOHO have the same MX records.)

It has 4 fields:-
Name:-
Mail-Server:- mx.zoho.com || mx2.zoho.com
Priority:- 1 (can be 10, 20, ...)
TTL:- 3600

TXT record:-

Stands for Text record. It allows us to add arbitrary text to domain name. Used to verify ownership of a website or verifying your domain for email automation services.

It has two fields:-
Name:- @ (website.com)
Content:- (can be any text used for verification)

NS record:-

Stands for Name Server record. It indicates the authoritative servers. There are mostly 2 nameservers. NS records are used to specify the authoritative name servers for a domain. TLD points to NS and NS give the A record of the requested website.

Glue record:-

These play a very good role when it comes to finding IP of a url. When the recursor receives query, it asks the root server which returns the TLD id. When it reaches the TLD, It would not know about the IP of the TLD, in this case, the root server gives additional information know as glue records that help in locating the TLD server which makes it faster in locating the IP of requested website.
