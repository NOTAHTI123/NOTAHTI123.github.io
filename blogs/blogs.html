DNS:-

     DNS, also known as Domain Name System, is a diverse process without which using the internet is not possible. DNS is used to translate the the url entered by the user to it's corresponding IP to let the web browser know where the user wants to navigate.


There are 4 DNS servers involved in loading a web page:-

      1. DNS recursor
      2. Root nameserver
      3. TLD nameserver
      4. Authoritative nameserver


1. DNS recursor:-

It can be thought of as a librarian who we ask to get a certain book. It receives the queries from client machines through applications such as a web browser. It is responsible for making additional requests in order to satisfy the client's DNS query.

2. Root nameserver:-

It is the first step in translating/resolving the human readable host names into IP addresses It can be thought of an index in library that points to racks of books.

3. TLD nameserver:-

The top level domain can be thought of the rack of books in a library. This nameserver is the next step in search for a specific IP address, it hosts the last portion of a domain name. For example in (google.com), the TLD server is "com".

4. Authoritative nameserver:-

It can be thought of as a dictionary in the rack of books. It is responsible for translating the domain name to it's corresponding IP address (if exists), and returns the IP to the DNS recursor.

DNS Lookup:-

There are 8 steps in a DNS lookup:-

1. A user types ‘example.com’ into a web browser and the query travels into the Internet and is received by a DNS recursive resolver.

2. The resolver then queries a DNS root nameserver (.).

3. The root server then responds to the resolver with the address of a Top Level Domain (TLD) DNS server (such as .com or .net), which stores the information for its domains. When searching for example.com, our request is pointed toward the .com TLD.

4. The resolver then makes a request to the .com TLD.

5. The TLD server then responds with the IP address of the domain’s nameserver, example.com.

6. Lastly, the recursive resolver sends a query to the domain’s nameserver.

7. The IP address for example.com is then returned to the resolver from the nameserver.

8. The DNS resolver then responds to the web browser with the IP address of the domain requested initially.
